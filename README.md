# Freya: God-Like Programming

Freya is an experimental programming language with a deliberately quirky and expressive syntax.

## About the Project

Freya started as a college team project: a simple IDE paired with a compiler built using ANTLR 4 and C#. The initial version supported basic features such as variable declarations, simple arithmetic operations, and output printing.

Development eventually stopped, but the idea didn’t.

With a deeper understanding of how regular expressions and language design work, this project is now being revived as a personal challenge: to build a fully functional compiler from scratch.

## Inspirations

The original syntax was inspired by scripting systems commonly found in video games. Over time, it also incorporated naming conventions from Cisco CLI commands, along with influences from Python and C.

To make things even more unusual, some Norwegian words were mixed into the language.

The result was a strange but interesting amalgam of styles-something unique, but not entirely coherent.

Freya is an attempt to refine that idea into a cleaner, more consistent language while preserving its original personality.

## Why "Freya"?

The project was originally called _JIP_, based on the initials of its creators. It was later renamed _Freya_, inspired by the song "Freya" by The Sword.

The name reflects the project’s philosophy: code should look elegant and beautiful on the surface, while hiding complexity beneath.

## Roadmap

1. Redefine the syntax from scratch, keeping only the most valuable ideas from the original project.
2. Choose a language to build the compiler (currently considering C, C++, or Pascal).
3. Develop two distinct versions of Freya’s syntax:
    - **Freya Classic**: A refined version of the original syntax, designed to be human-readable and practical.
    - **Freya Machine Girl**: A more experimental syntax inspired by esoteric programming languages, designed to feel closer to how a machine might "think."
